Course Overview

In this course, you will learn how to solve problems in Java while increasing your Java programming skills through hands-on programming of apps and games. We will focus on working together to increase your proficiency for a set of Learning Objectives (LOs), advancing through 4 levels of mastery

Intern → Junior Developer →  Middle Developer → Senior Developer level.

Learning Outcomes

There are 10 Learning Outcomes (LOs) for this course. When you successfully complete this course you will be able to...

  1. Employ design principles of Object Oriented Programming
  2. Construct programs using single and multidimensional arrays
  3. Construct programs using objects and classes
  4. Construct programs using inheritance and polymorphism, including abstract classes and interfaces
  5. Construct programs using exception handling
  6. Construct programs using text file Input/Output
  7. Construct programs using graphical user interfaces utilizing event-driven programming
  8. Construct programs using generic collections and data structures
  9. Construct programs using recursion
  10. Construct programs using multiple threads

There are four levels of proficiency for each LO. In the diagram below, you can see how each level of proficiency leads to the next, and what you need to demonstrate to reach the next level. Most modules will guide you through the first two levels for a single LO, with multiple activities for practice at each level. You can use these activities to improve and to demonstrate your proficiency before moving on to the next level. During the class, there will be a project where you will have an opportunity to demonstrate mastery.

Some of the LOs were covered briefly in CISC 190, but you may have decided to dive in deeper and therefore you are already at a higher level of proficiency for a particular LO. That’s okay - you can choose which activities you will complete to demonstrate proficiency - you can skip the early levels and jump right in at the Middle Developer level if you want!

Common Student Questions

How does this class work?

The class is designed to provide you with multiple learning opportunities, including discussing the material with your classmates and practicing applying what you’ve learned to fun problems.

To succeed in this class, I recommend that you complete the reading and attempt the Level 1 activities early in the week(s) assigned to the module. Actively engage in discussion and code-writing activities. Work with your classmates to help improve both your proficiency and theirs! When you are ready, create and submit an explanation video to demonstrate understanding and Level 3 proficiency. A class project provides an opportunity to demonstrate Level 4 mastery.

Plan to work a little each day, and to spend 6-10 hours outside of class time each week working on this course.

How do you assign letter grades for the class?

For each LO, there are a variety of activities to demonstrate proficiency levels. Each activity is aligned with a particular level, and is graded on a Satisfactory/Not Yet basis with feedback provided (Canvas will display as complete/incomplete). Successful completion of activities for a particular proficiency level mean that you have achieved that level for the LO.

Proficiency Level Demonstrations

How do I demonstrate...

Intern

Level 1

Take a multiple choice quiz on Canvas to demonstrate that you can read code.

Junior Developer

Level 2

Write code to make a test program run successfully.

Middle Developer

Level 3

In a screencast style video with your voice, explain how a concept works in code, and when it should be used.

Senior Developer

Level 4

Use multiple concepts in a project, then make a video explaining where, why, and how each concept contributes to the overall solution.

 

At the end of the term, the count of Level scores will be translated to grades. The highest rating for proficiency earned in each LO will be used in determining grades.

 

Learning Outcome to Grade Translation
Grade LO Levels
A Proficiency Level 4 on at least 8 of the 10 LOs, and no Level 0s or Level 1s
B Proficiency Level 3 or higher on at least 8 of the 10 LOs, and no Level 0s or Level 1s
C Proficiency Level 2 or higher on all LOs
D Proficiency Level 1 or higher on at least 5 of the 10 LOs
F Proficiency Level 0 (no work submitted) on more than 5 LOs.

When are assignments due?

Each activity in a module has a due date to help keep you on track; I expect that you will submit a reasonable attempt on time, even if it is not yet complete. If you have not successfully demonstrated Junior Developer (Level 2) proficiency or higher for a LO by the last due date of the Module in which it is covered, that is a sign you may need extra help. To ensure your success in the course, I will ask you to attend office hours and/or tutoring until you are caught up.

Modules are released as the course progresses so that we move through the class material as a cohort. But, everyone learns at a different pace. Work for any assignment will be accepted for full credit through Friday, May 24.

Campus administrative deadlines that you should know:

  • Friday, February 9 (Add/Drop Deadline). Successfully complete through the end of Module 1 or you may be dropped from the course. In my experience with students in previous terms, you are not likely to successfully complete the course if you haven’t gotten started by the add/drop deadline.
  • Friday April 12 (Withdraw Deadline). Demonstrate a reasonable attempt for at least Junior Developer Level on Modules 1-5 or you may be withdrawn from the course. In my experience with students in previous terms, you are not likely to successfully complete the course if you have not been able to keep up.

Can I improve my grade on an assignment?

Yes! If you do not successfully complete an assignment, you should rework it and resubmit when it is improved. You may want to review the activities at one level lower than the activity you’re working on to ensure that you learned everything you could at that level. Resubmitted work can only improve your grade, never lower it.

What is the collaboration policy?

Learning is a community activity. I encourage you to work with classmates and form virtual study groups to support your learning. Use pair programming, especially at the Junior Developer level. Pairs are also encouraged for projects. You are responsible for your own learning; when collaborating with classmates, if you don’t understand something, make sure to ask! Videos for Middle and Senior Developer levels must be created independently by each student (e.g. don’t share a script with your partner), but will frequently reference the same code (that you worked together to write).

All submitted code must have the names of all people who helped you with the code (regardless of whether the person assisting you is enrolled in this course). Additionally, if you borrowed source code, or gained inspiration from, an online source, that source must be clearly cited in the code comments.

What skills am I learning in this course?

In addition to learning Java programming, you will also have the opportunity to practice and refine Employable Skills that are in demand by employers. Did you know that employers are often less concerned about what programming languages you know (these skills are easy to learn on the job), and are more concerned about collaboration and communication skills? Here is a list of some of the skills that you will be able to put on your resume: critical thinking, problem solving, analytical skills, teamwork/collaboration, troubleshooting, research, planning, creativity, resilience/perseverance, written communication, lifelong learning.

Help!  I’m stuck!  What should I do?

Great question!  I never want you to feel “stuck” for too long before asking for help. Here are some suggestions for places to find help:

  • Ask questions! Use Discord and the class Q&A forum.
  • Watch your classmates’ videos that demonstrate Middle and Senior Developer proficiency.
  • Seek out free online tutoring from Mesa.
  • Form a virtual study group with your classmates.
  • You are always welcome to come to office hours or to e-mail me using the Canvas Inbox at any time.

Course Description:

This course is an intermediate level study of the Java programming language. Topics include single and multidimensional arrays; objects and classes; object-oriented programming; inheritance and polymorphism; exception handling and text input/output (I/O); abstract classes and interfaces; graphical user interfaces (GUIs); recursion; concurrency; and generic collections and data structures, such as linked lists, queues, and stacks. This course is intended for students majoring in computer and information sciences or anyone interested in learning more about the Java programming language.

Prerequisite

CISC 190 with grade of “C” or better, or equivalent.

Course Learning Outcomes:

Students will be able to:

  1. Employ design principles of object-oriented programming
  2. Construct programs utilizing inheritance and polymorphism, including abstract classes and interfaces
  3. Construct programs utilizing exception handling 
  4. Construct programs utilizing graphical user interfaces utilizing event-driven programming

Student Learning Objectives:

Upon successful completion of the course the student will be able to:

  1. Employ design principles of object-oriented programming.
  2. Utilize the syntax and semantics of an object-oriented language in the development of software.
  3. Effectively use software development tools including libraries, compilers, editors, debuggers, and version control systems.
  4. Construct programs utilizing single and multidimensional arrays; objects and classes; object-oriented programming; inheritance and polymorphism; exception handling and text I/O; abstract classes and interfaces; GUI; recursion; concurrency, and generic collections and data structures, such as linked lists, queues, and stacks.
  5. Systematically design and implement intermediate-level Java programs from given specifications, such as problem descriptions, user stories, design criteria, input/output scenarios, functional or unit tests, pseudo code, modeling language specifications or diagrams.
  6. Articulate the purpose and working of a program, and its language constructs and data structures, as included in this course, such as through diagrams (for instance, flowcharts, Unified Modelling Language (UML) diagrams or entity-relationship (ER) diagrams), code comments, screencast videos, interactive computing notebooks (for instance, Jupyter notebooks), or written or oral communication.
  7. Document external interfaces of intermediate-level Java methods (the application programming interface (API); for instance, Javadoc), and the inner working of methods in code comments.
  8. Systematically test and debug intermediate-level Java programs by interactively entering arguments, utilizing functional or unit tests, running programs in a test environment, or using a debug environment
    (for instance, a debugger in an integrated development environment (IDE)).
  9. Develop intermediate-level Java programs using industry recognized tools, such as code editors, command lines (Read, execute, print loop, REPL), emulators, or integrated development environments (IDE).
  10. Create a fully functional, interactive program incorporating a substantial set of topics from the course.
