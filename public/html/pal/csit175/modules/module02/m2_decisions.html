Making Decisions

Boolean Expressions

Sequential processing consists of passing an ordered list of instructions through a processor. Do step 1, then step 2, step 3, and step 4, and so on.

Often, in any sequential list of instructions there will be conditional execution. If it's raining outside, then find an umbrella. If the dog is scratching at the door, then open the door and let her go outside. If it is your grandmother's birthday, then we will be joining her for dinner at her favorite restaurant.

Conditional execution involves evaluating a boolean expression. This is a binary expression. It can only evaluate to

  • True or False,
  • Yes or No,
  • On or Off.

 

Here are some examples of boolean expressions.

  • 5 > 3
  • 10 == 10
  • x < y
  • a >= b
  • word1 == word2
  • is_sunny and is_warm
  • not is_raining
  • hour_of_day <= 12
  • temperature > 75 or humidity < 60
  • count != 0
  • not (x == y)
  • is_weekend or is_holiday
  • age >= 18 and age < 21
  • not (is_working_day and is_busy)
  • username == "admin" and password == "secure123"
  • not is_empty or is_full
  • grade == 'A' or grade == 'B'
  • points >= 90 and not is_cheating
  • not (is_morning and is_weekday)
  • is_valid and not is_expired

Python Conditional Statements: if, elif, else

In Python, the if statement is used to conditionally execute a block of code. The keyword if is followed by a boolean expression and a colon, then the conditional code to be executed is indented (typically 4 spaces) and then processing returns from the code block by un-indenting to the same column that it started.

For example: 

            if TODAY == "Friday":
                print("Weekend is almost here!")
                hours_worked += 8
            print(TODAY)

 

In Python, conditional statements are used to execute different blocks of code based on certain conditions. The key components for this are the if, elif (else if), and else statements.

if Statement:

The if statement is used to execute a block of code if a specified condition evaluates to True. Here's the basic syntax: 

if condition:
    # Code to be executed if the condition is True

elif Statement:

The elif statement allows you to check additional conditions if the previous if or elif conditions are not met. It stands for "else if." Here's how it works: 

if condition:
    # Code to be executed if the condition is True
elif another_condition:
    # Code to be executed if another_condition is True

else Statement:

The else statement is used to execute a block of code when none of the previous conditions (in if or elif) are true. It acts as a fallback. Here's the syntax: 

if condition:
    # Code to be executed if the condition is True
elif another_condition:
    # Code to be executed if another_condition is True
else:
    # Code to be executed if none of the above conditions are True

Together, these statements provide a way to create structured and controlled flow in your Python programs, allowing for different code paths based on varying conditions.

Notice that the elif statement is not just another if statement. It relies on the first if condition to be evaluated as False or it will never be evaluated.

Likewise, any else statements won't execute unless all of the previous if and elif boolean conditions have evaluated as false.

Remember that not all if statements will need an else statement.

Finally, remember that all if and elif statements must have a corresponding boolean expression, and an else statement never will.

 

Indenting

As forgiving and simple as the Python programming language tends to be, be aware that it is very particular about indentation. Each code block is identified and distinguished by the level of indentation, and returning to the previous indentation level after the code block is executed. By default, this should always be 4 spaces. In fact, the language isn't particular about how many spaces as long as it's consistent and always the same.

As a general rule, every popular code editor will translate the Tab character to a fixed number of spaces. It's a good idea to get in the habit of using the [Tab] key to enter the correct number of spaces, and not the spacebar. The [Tab] key won't get it wrong.
