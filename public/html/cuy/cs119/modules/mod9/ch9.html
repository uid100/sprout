Chapter Notes

Event-Driven Programming

Chapter 9 introduces the concept of event-driven code.

You have seen that an application can be started and sequentially execute lines of code logically one after another. But often we need to write an application that will depend on user inputs, not only for data, but to select which tasks to perform and in which order. Graphical User Interfaces (GUIs) are a good example of event-driven program exection.
