In chapter 6, we move into user defined methods. In other languages, these are often referred to as functions and/or procedures. In languages that have both, functions return a value; procedures do not. In Python, we'll use the term method.

Overloading - not supported in Python

Failed overload

In Python, overloaded methods are not supported. This demo shows an attempt to do it and how it fails.

#!/usr/bin/env python3

# Cuyamaca College CS-119
# Misc method code for the chapter 6 notes

# "Global" constant that can accessed anywhere in this module
RAISE_PCT = 1.1

# Python main() method
def main():
    # attempt at calling "overloaded mehods" in Python
    # WARNING: You will get errors running this!
    new_salary1 = calc_new_salary(1000)
    new_salary2 = calc_new_salary(1000, 2.5)
    print("New salary 1 = " + str(new_salary1) + " New salary 2 = " + str(new_salary2))


# Attempt at method overloading in Python
# WARNING! This does not work!

def calc_new_salary(old_salary):
    return old_salary * RAISE_PCT

def calc_new_salary(old_salary, raise_pct):
    return old_salary * raise_pct


# invoke the main method - note this is at the end
# note the double underscore __ and indent 4 spaces
if __name__ == "__main__":
    main()

Workaround

Python does allow using optional parameters as a "sort of" way to overload methods

#!/usr/bin/env python3

# Cuyamaca College CS-119
# Misc method code for the chapter 6 notes
# Ch6MiscDemoCodeMethodOverloadKindaSorta.py

# "Global" constant that can accessed anywhere in this module
RAISE_PCT = 1.1

# Python main() method
def main():
    # attempt at calling "overloaded mehods" in Python
    new_salary1 = calc_new_salary(1000)
    new_salary2 = calc_new_salary(1000, 2.5)
    print("New salary 1 = " + str(new_salary1) + "\nNew salary 2 = " + str(new_salary2))


# Attempt at achieving method overloading in Python by using optional parameters.
# This is kind of a cheesy back handed approach. Whether or not this is
# a great idea is arguable...
# Code is based on a demo found at: https://pythonspot.com/method-overloading/

def calc_new_salary(old_salary, raise_pct=None):
    if raise_pct is not None:
        return old_salary * raise_pct
    else:
        # Note we use our global constant
        return old_salary * RAISE_PCT



# invoke the main method - note this is at the end
# note the double underscore __ and indent 4 spaces
if __name__ == "__main__":
    main()

Variable Scope

Variables only have meaning where they are defined (in scope).

#!/usr/bin/env python3

# Cuyamaca College CS-119
# Misc method code for the chapter 6 notes

# "Global" constant that can accessed anywhere in this module
RAISE_PCT = 1.1

# Python main() method
def main():
    
    new_salary1 = calc_new_salary1(1000)
    new_salary2 = calc_new_salary2(1000, 2.5)
    new_salary3 = calc_new_salary3(1000)
    print("New salary 1 = " + str(new_salary1) + "\nNew salary 2 = " + str(new_salary2) + "\n")
    print("New salary 3 = " + str(new_salary3))


# Method overloading in Python doesn't work sooo... create 3 different methods

def calc_new_salary1(old_salary):
    return old_salary * RAISE_PCT # uses the "global" constant

def calc_new_salary2(old_salary, raise_pct):
    return old_salary * raise_pct

def calc_new_salary3(old_salary):
    LOCAL_RAISE_PCT = 1.5 # local constant to calcSalary3()
    return old_salary * LOCAL_RAISE_PCT

# invoke the main method - note this is at the end
# note the double underscore __ and indent 4 spaces
if __name__ == "__main__":
    main()
