I know you've heard me say it before. I really do like writing code. I enjoy the challenge, I enjoy the satisfaction when it's done!

Of course done can be subjective too. When you write code, usually you start with the framework. Get something running first then satisfy frequirements until they are all met. But even then, the code should be revisited to optimize and clarify things. Look it over and decide what could be written better.

And testing! Always test while you're writing your code. There may be more testing later, but the code should be pretty thorougly tested before it reaches 'formal' testing.
