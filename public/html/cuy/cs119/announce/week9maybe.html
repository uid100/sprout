Entities, things, stuff, objects.. this thing, that thing... 

Considerations of object-oriented programming starts with the objects.  Identify the entities first, then describe them by their features and behaviors. Focus on the thing first, the syntax will follow.

 

The lab exercises in this course are relatively simple (not complex). Example code, followed by a chance to practice and apply the same concept in a (sometimes) different way. In most cases, there is a more complex problem statement. In these cases, it's usually best practice to identify what the entities are first, and how they interact. This makes the code easier to right and easier to modularize and encapsulate.

 

An object is an instance of a class.  Write the class, then make instances from the class. If you need to change the object, consider if the class needs to change first.
